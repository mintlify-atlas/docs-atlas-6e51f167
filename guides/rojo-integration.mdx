rbx-tsx is designed to work seamlessly with [Rojo](https://rojo.space/), the industry-standard tool for syncing files between your filesystem and Roblox Studio.

## Quick Start

Scaffold a new rbx-tsx + Rojo project:

```bash
rbx-tsx init my-game
cd my-game
```

This creates:
- `default.project.json` — Rojo project configuration
- `wally.toml` — Roblox package dependencies (React, ReactRoblox)
- `tsconfig.json` — TypeScript configuration
- `src/` — Source TypeScript/TSX files
- `out/` — Compiled Luau output

## Rojo Project Structure

**Typical project layout:**

```
my-game/
├── default.project.json    # Rojo configuration
├── wally.toml              # Wally dependencies
├── tsconfig.json           # TypeScript config
├── package.json            # npm dependencies
├── src/                    # TypeScript source files
│   ├── client/
│   │   ├── index.client.tsx
│   │   └── App.tsx
│   ├── server/
│   │   └── index.server.ts
│   └── shared/
│       └── utils.ts
├── out/                    # Compiled Luau output
│   ├── client/
│   │   ├── init.luau
│   │   └── App.luau
│   ├── server/
│   │   └── init.luau
│   └── shared/
│       └── utils.luau
└── Packages/               # Wally packages (React, etc.)
```

## Rojo Configuration

**default.project.json:**

```json
{
  "name": "my-game",
  "tree": {
    "$className": "DataModel",
    "StarterPlayer": {
      "$className": "StarterPlayer",
      "StarterPlayerScripts": {
        "$className": "StarterPlayerScripts",
        "Client": {
          "$path": "out/client"
        }
      }
    },
    "ServerScriptService": {
      "$className": "ServerScriptService",
      "Server": {
        "$path": "out/server"
      }
    },
    "ReplicatedStorage": {
      "$className": "ReplicatedStorage",
      "Shared": {
        "$path": "out/shared"
      },
      "Packages": {
        "$path": "Packages"
      }
    }
  }
}
```

<Note>
rbx-tsx automatically detects `default.project.json` when compiling directories. No additional configuration needed.
</Note>

## Wally Integration

**wally.toml:**

```toml
[package]
name = "developer/my-game"
version = "0.1.0"
registry = "https://github.com/UpliftGames/wally-index"
realm = "shared"

[dependencies]
React = "jsdotlua/react@17.1.0"
ReactRoblox = "jsdotlua/react-roblox@17.1.0"
```

Install dependencies:

```bash
wally install
```

This creates a `Packages/` directory with React and ReactRoblox.

<Warning>
Run `wally install` before compiling to ensure React dependencies are available in Roblox.
</Warning>

## Compilation Workflow

### 1. Compile TypeScript to Luau

```bash
rbx-tsx compile src/ -o out/
```

Or with CSS support:

```bash
rbx-tsx compile src/ -o out/ --css
```

### 2. Sync to Roblox Studio

```bash
rojo serve
```

Then in Roblox Studio, click the **Rojo plugin** button to connect.

### 3. Watch Mode (Development)

```bash
# Terminal 1: Watch and compile on changes
rbx-tsx watch src/ -o out/

# Terminal 2: Sync to Roblox Studio
rojo serve
```

Changes to `.tsx` files will auto-recompile and sync to Studio.

## Path Alias Resolution

rbx-tsx reads `default.project.json` to resolve cross-boundary imports.

**Example:**

```json
// default.project.json
{
  "tree": {
    "ReplicatedStorage": {
      "Shared": { "$path": "out/shared" },
      "Components": { "$path": "out/components" }
    },
    "StarterPlayer": {
      "StarterPlayerScripts": {
        "Client": { "$path": "out/client" }
      }
    }
  }
}
```

**Import from client to shared:**

```tsx
// src/client/App.tsx
import { formatTime } from "../../shared/utils";
```

**Compiled:**

```lua
-- Automatically resolves to the Roblox path from default.project.json
local _module_utils = require(game:GetService("ReplicatedStorage").Shared.utils)
local formatTime = _module_utils.formatTime
```

rbx-tsx detects that `out/shared` maps to `ReplicatedStorage.Shared` and generates the correct `require()` path.

<Note>
Path alias resolution works automatically when compiling directories. No configuration needed.
</Note>

## File Naming Conventions

rbx-tsx follows Rojo's file naming conventions:

| Source File | Compiled Output | Rojo Behavior |
|-------------|-----------------|---------------|
| `index.client.tsx` | `init.client.luau` | LocalScript in StarterPlayerScripts |
| `index.server.ts` | `init.server.luau` | Script in ServerScriptService |
| `index.tsx` | `init.luau` | ModuleScript (folder module) |
| `App.tsx` | `App.luau` | ModuleScript |
| `types.d.ts` | _(skipped)_ | Type declarations are not compiled |

<Warning>
**Important:** Files ending in `.client.tsx` or `.server.ts` become `LocalScript` or `Script` instances, not `ModuleScript`. Use them for entry points only.
</Warning>

## Example: Client Entry Point

**src/client/index.client.tsx:**

```tsx
import React from "react";
import { createRoot } from "react-roblox";
import { Players } from "@rbx-services";
import App from "./App";

const playerGui = Players.LocalPlayer.WaitForChild("PlayerGui") as PlayerGui;

const screenGui = new Instance("ScreenGui");
screenGui.Name = "App";
screenGui.Parent = playerGui;
screenGui.ResetOnSpawn = false;
screenGui.ZIndexBehavior = Enum.ZIndexBehavior.Sibling;

const root = createRoot(screenGui);
root.render(<App />);
```

**Compiled out/client/init.client.luau:**

```lua
local React = require(game:GetService("ReplicatedStorage").Packages.React)
local ReactRoblox = require(game:GetService("ReplicatedStorage").Packages.ReactRoblox)
local createRoot = ReactRoblox.createRoot
local Players = game:GetService("Players")
local App = require(script.App)

local playerGui = Players.LocalPlayer:WaitForChild("PlayerGui")

local screenGui = Instance.new("ScreenGui")
screenGui.Name = "App"
screenGui.Parent = playerGui
screenGui.ResetOnSpawn = false
screenGui.ZIndexBehavior = Enum.ZIndexBehavior.Sibling

local root = createRoot(screenGui)
root:render(React.createElement(App, nil))
```

Rojo syncs this to `StarterPlayer > StarterPlayerScripts > Client` as a `LocalScript`.

## Custom Require Paths

Override default React/ReactRoblox paths:

```bash
rbx-tsx compile src/ -o out/ \
  --react-path "ServerStorage.Libs.React" \
  --react-roblox-path "ServerStorage.Libs.ReactRoblox"
```

**Compiled:**

```lua
local React = require(game:GetService("ServerStorage").Libs.React)
local ReactRoblox = require(game:GetService("ServerStorage").Libs.ReactRoblox)
```

## Build Scripts

Add build scripts to `package.json`:

```json
{
  "scripts": {
    "build": "rbx-tsx compile src -o out",
    "build:css": "rbx-tsx compile src -o out --css",
    "watch": "rbx-tsx watch src -o out",
    "check": "rbx-tsx check src"
  }
}
```

Run with:

```bash
npm run build
npm run watch
```

## Production Build

For production, compile with strict mode:

```bash
rbx-tsx compile src/ -o out/ --strict
```

This treats warnings as errors and fails the build on any issues.

## Full Development Workflow

1. **Initialize project:**
   ```bash
   rbx-tsx init my-game
   cd my-game
   npm install
   wally install
   ```

2. **Start development:**
   ```bash
   # Terminal 1: Watch and compile
   rbx-tsx watch src/ -o out/

   # Terminal 2: Sync to Roblox Studio
   rojo serve
   ```

3. **Connect Roblox Studio:**
   - Open Roblox Studio
   - Install the [Rojo plugin](https://rojo.space/docs/v7/getting-started/installation/)
   - Click the Rojo plugin button and connect to `localhost:34872`

4. **Make changes:**
   - Edit `.tsx` files in `src/`
   - rbx-tsx auto-compiles to `out/`
   - Rojo auto-syncs to Roblox Studio

5. **Test in Roblox Studio:**
   - Press F5 to playtest
   - Changes hot-reload automatically

## Troubleshooting

### "Module not found" errors in Roblox

**Cause:** Rojo paths don't match compiled `require()` paths.

**Fix:** Ensure `default.project.json` correctly maps output directories:

```json
{
  "ReplicatedStorage": {
    "Packages": { "$path": "Packages" }  // Must match Wally output
  }
}
```

### "React is not a valid member" errors

**Cause:** Wally packages not installed.

**Fix:**

```bash
wally install
rojo serve  # Restart Rojo after installing packages
```

### Path aliases not resolving

**Cause:** `default.project.json` not found or output directory mismatch.

**Fix:** Ensure you're compiling from a directory containing `default.project.json`, and that the output path matches the `$path` values in the Rojo config.

### Changes not syncing to Studio

**Cause:** Rojo not watching the correct directory.

**Fix:** Restart `rojo serve` and reconnect from Studio.

## Advanced: Multi-Place Projects

For multi-place games, create separate Rojo configs:

**lobby.project.json:**
```json
{
  "name": "Lobby",
  "tree": {
    "ReplicatedStorage": {
      "Shared": { "$path": "out/shared" },
      "Lobby": { "$path": "out/lobby" }
    }
  }
}
```

**game.project.json:**
```json
{
  "name": "Game",
  "tree": {
    "ReplicatedStorage": {
      "Shared": { "$path": "out/shared" },
      "Game": { "$path": "out/game" }
    }
  }
}
```

Serve with:

```bash
rojo serve lobby.project.json
rojo serve game.project.json --port 34873
```

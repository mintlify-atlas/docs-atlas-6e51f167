Scaffold a new Roblox project with rbx-tsx, React, and Rojo configuration.

## Usage

```bash
rbx-tsx init [directory] [options]
```

### Arguments

<ParamField path="directory" type="string">
  Project directory to create.
  
  **Default**: Current directory (`.`)
  
  If the directory doesn't exist, it will be created.
</ParamField>

## Options

<ParamField path="-n, --name" type="string">
  Project name
  
  **Default**: Directory name
  
  Used in:
  - `package.json` name field
  - `wally.toml` package name
  - `default.project.json` project name
</ParamField>

## Examples

<CodeGroup>

```bash Initialize in current directory
rbx-tsx init
```

```bash Create new project directory
rbx-tsx init my-app
```

```bash Custom project name
rbx-tsx init my-app --name "My Awesome Game"
```

```bash Specify name with flag
rbx-tsx init . -n my-game
```

</CodeGroup>

## What Gets Created

The `init` command scaffolds a complete Roblox project structure:

### Directory Structure

```
my-app/
├── src/
│   ├── client/
│   │   ├── index.client.tsx    # Client entry point
│   │   └── App.tsx              # Root React component
│   ├── server/
│   │   └── index.server.ts      # Server entry point
│   └── shared/                  # Shared code (empty)
├── out/                         # Compiled Luau output
├── Packages/                    # Wally packages (after wally install)
├── package.json                 # npm dependencies
├── tsconfig.json                # TypeScript configuration
├── wally.toml                   # Roblox package manager
└── default.project.json         # Rojo project file
```

    import { Players } from "@rbx-services";

    const screenGui = new Instance("ScreenGui");
    screenGui.Name = "App";
    screenGui.Parent = playerGui;
    screenGui.ResetOnSpawn = false;

    root.render(<App />);

### If Files Already Exist

Existing files are skipped:

```
  create package.json
  skip   tsconfig.json (already exists)
  create wally.toml
  ...
```

### If Dependencies Fail

If npm or wally installation fails, instructions are provided:

```
Could not install npm dependencies. Run 'bun install' or 'npm install' manually.

Could not run wally. Install it from https://github.com/UpliftGames/wally
Then run: wally install
```

## Post-Init Steps

### 1. Install Dependencies

If automatic installation failed:

```bash
cd my-app
bun install  # or npm install
wally install
```

### 2. Compile TypeScript

```bash
rbx-tsx compile src/ -o out/
```

Or use watch mode:

```bash
rbx-tsx watch src/ -o out/
```

### 3. Open in Roblox Studio

```bash
rojo serve
```

Then in Roblox Studio:
- Go to Plugins → Rojo
- Click "Connect"
- Your project will sync to the game

### 4. Start Developing

Edit `src/client/App.tsx` and see changes live in Roblox Studio.

## Installed Packages

### npm (for TypeScript)

- `rbx-tsx` - TypeScript to Luau compiler with type definitions

### Wally (for Roblox)

- `React` (`jsdotlua/react@17.1.0`) - React for Luau
- `ReactRoblox` (`jsdotlua/react-roblox@17.1.0`) - React renderer for Roblox

Packages are installed to:
- `node_modules/` (npm)
- `Packages/` (Wally)

## Customization

After initialization, you can customize:

### Add More Dependencies

**npm** (for type definitions):
```bash
npm install --save-dev @rbxts/types
```

**Wally** (for Roblox packages):
```toml
[dependencies]
Roact = "roblox/roact@1.4.4"
```

Then run `wally install`.

### Modify Rojo Configuration

Edit `default.project.json` to change:
- Output paths
- Instance tree structure
- Build targets

### Add Build Scripts

Edit `package.json`:
```json
{
  "scripts": {
    "build": "rbx-tsx compile src/ -o out/",
    "watch": "rbx-tsx watch src/ -o out/",
    "check": "rbx-tsx check src/ --strict"
  }
}
```

Run with:
```bash
npm run build
npm run watch
npm run check
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Permission denied">
    - Run with appropriate permissions
    - Check directory write access
    - On Windows, try running terminal as administrator
  </Accordion>
  
  <Accordion title="npm install fails">
    - Ensure Node.js is installed: `node --version`
    - Try using `bun install` instead
    - Check internet connection
    - Clear npm cache: `npm cache clean --force`
  </Accordion>
  
  <Accordion title="wally install fails">
    - Ensure Wally is installed: `wally --version`
    - Install Wally from [GitHub](https://github.com/UpliftGames/wally)
    - Check `wally.toml` syntax
  </Accordion>
  
  <Accordion title="Directory already exists">
    The `init` command will:
    - Use existing directory
    - Skip files that already exist
    - Only create missing files
    
    To start fresh, remove the directory first.
  </Accordion>
</AccordionGroup>

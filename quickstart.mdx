# Quickstart

This tutorial walks you through creating a Counter component, compiling it to Luau, and understanding the output. You'll learn the basics of rbx-tsx in under 5 minutes.

## Prerequisites

Make sure you have rbx-tsx installed. If not, see the [Installation](/installation) guide.

## Create Your First Component

<Steps>
  <Step title="Create a TypeScript file">
    Create a new file called `Counter.tsx`:

    ```tsx Counter.tsx
    import React, { useState, useCallback } from "react";

    interface CounterProps {
      label: string;
      initialCount?: number;
    }

    export default function Counter({ label, initialCount = 0 }: CounterProps) {
      const [count, setCount] = useState(initialCount);

      const increment = useCallback(() => {
        setCount((c) => c + 1);
      }, []);

      return (
        <div className="counter">
          <h1>{label}</h1>
          <span>Count: {count}</span>
          <button onClick={increment}>+</button>
        </div>
      );
    }
    ```

    This component demonstrates:
    - **TypeScript interfaces** for type-safe props
    - **React hooks** (`useState`, `useCallback`)
    - **JSX** with HTML-like elements
    - **Optional props** with default values
  </Step>

  <Step title="Compile to Luau">
    Run the compiler:

    ```bash
    rbx-tsx compile Counter.tsx -o Counter.luau
    ```

    This creates `Counter.luau` with the compiled output.
  </Step>

  <Step title="Examine the output">
    Open `Counter.luau` to see the compiled Luau:

    ```lua Counter.luau
    local React = require(game:GetService("ReplicatedStorage").Packages.React)
    local useState = React.useState
    local useCallback = React.useCallback

    type CounterProps = {
        label: string,
        initialCount: number?,
    }

    local function Counter(props: CounterProps)
        local label = props.label
        local initialCount = if props.initialCount ~= nil then props.initialCount else 0
        local count, setCount = useState(initialCount)
        local increment = useCallback(function()
            setCount(function(c) return c + 1 end)
        end, {})

        return React.createElement("Frame", { [React.Tag] = "counter" }, {
            H1 = React.createElement("TextLabel", { Text = label }),
            Span = React.createElement("TextLabel", { Text = `Count: {count}` }),
            Button = React.createElement("TextButton", {
                [React.Event.Activated] = increment,
                Text = "+",
            }),
        })
    end

    return Counter
    ```

    Notice how rbx-tsx:
    - Preserves **full type annotations** (`CounterProps`, `label: string`, etc.)
    - Converts JSX to **React.createElement calls**
    - Maps `<div>` to `Frame`, `<h1>` to `TextLabel`, `<button>` to `TextButton`
    - Handles `className` as `React.Tag`
    - Converts `onClick` to `React.Event.Activated`
    - Maintains **clean, readable** structure
  </Step>
</Steps>

## Understanding the Compilation

Let's break down what rbx-tsx did:

### Type Preservation

TypeScript interfaces become Luau type definitions:

<CodeGroup>

```tsx TypeScript
interface CounterProps {
  label: string;
  initialCount?: number;
}
```

```lua Luau
type CounterProps = {
    label: string,
    initialCount: number?,
}
```

</CodeGroup>

### React Hooks

Hooks are imported from the react-lua package:

<CodeGroup>

```tsx TypeScript
import React, { useState, useCallback } from "react";

const [count, setCount] = useState(initialCount);
```

```lua Luau
local React = require(game:GetService("ReplicatedStorage").Packages.React)
local useState = React.useState
local useCallback = React.useCallback

local count, setCount = useState(initialCount)
```

</CodeGroup>

### JSX to Elements

HTML-like JSX compiles to Roblox GUI classes:

<CodeGroup>

```tsx TypeScript
<div className="counter">
  <h1>{label}</h1>
  <button onClick={increment}>+</button>
</div>
```

```lua Luau
React.createElement("Frame", { [React.Tag] = "counter" }, {
    H1 = React.createElement("TextLabel", { Text = label }),
    Button = React.createElement("TextButton", {
        [React.Event.Activated] = increment,
        Text = "+",
    }),
})
```

</CodeGroup>

<Tip>
  See the [Element Mapping](/guides/element-mapping) guide for a complete list of HTML to Roblox class mappings.
</Tip>

## Compile a Directory

Instead of compiling single files, you can compile entire directories:

```bash
rbx-tsx compile src/ -o out/
```

This compiles all `.ts` and `.tsx` files in `src/` to the `out/` directory, maintaining the folder structure.

## Watch Mode

During development, use watch mode to automatically recompile when files change:

```bash
rbx-tsx watch src/ -o out/
```

<Note>
  Watch mode continues running until you stop it with `Ctrl+C`. It monitors your source files and recompiles on save.
</Note>

## Type Checking

To check for errors without emitting files:

```bash
rbx-tsx check Counter.tsx
```

This runs the compiler and reports warnings and errors without creating output files.
